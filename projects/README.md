## How To Present A Project In Interviews

Use this structure:

1. Problem: what user pain did you solve?
2. Architecture: what components did you build?
3. Tradeoffs: what design decisions mattered?
4. Evaluation: how did you know it worked?
5. Failure cases: what broke and how did you fix it?
6. Impact: what improved?
