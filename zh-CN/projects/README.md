## 面试中如何介绍项目

使用这个结构：

1. Problem：解决了什么用户痛点？
2. Architecture：构建了哪些组件？
3. Tradeoffs：哪些设计决策最重要？
4. Evaluation：如何证明它有效？
5. Failure cases：哪里失败过，如何修复？
6. Impact：带来了什么改进？
