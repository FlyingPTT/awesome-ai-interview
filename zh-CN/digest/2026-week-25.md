# 每周 AI 面试 Digest：2026 Week 25

第一期 digest 先作为仓库维护记录，而不是新闻周报。它总结初版 handbook 发布后，候选人应该重点关注什么。

## Top Trends

- AI Agent 工程正在成为核心面试主题，尤其是工具使用、状态、验证和安全。
- RAG 面试正在从“向量数据库基础”转向 evaluation、reranking、access control 和生产 debugging。
- 可运行作品集项目比静态笔记更有说服力。

## 模型与产品更新

| Item | 为什么重要 | 面试角度 |
| --- | --- | --- |
| Coding agents | 暴露真实的 tool-use 和 repository-editing 问题。 | 讨论 sandboxing、tests、patch review 和 audit logs。 |
| Evaluation harnesses | 团队需要为 prompt 和 model changes 建立 regression gates。 | 讨论 rubrics、deterministic checks、judge calibration、latency 和 cost。 |
| Agent workflows | 生产 Agent 需要显式控制流。 | 讨论 state machines、tool schemas、stop conditions 和 human approval。 |

## 值得读的论文方向

| Paper Area | Topic | Takeaway |
| --- | --- | --- |
| RAG | Retrieval and faithfulness | Retrieval 和 generation 要分开评估。 |
| Agents | Tool use and planning | Tool schema design 是 Agent 质量的一部分。 |
| Inference | Serving optimization | KV Cache、batching 和 routing 是系统设计主题。 |

## 新增面试题

- 如何设计一个 minimal agent loop？
- 如何构建 LLM evaluation harness？
- RAG 中如何区分 retrieval failure 和 generation failure？
- 如何防止 Agent 运行不安全工具？

## 项目 Ideas

- 给 minimal agent framework 增加 human approval hooks。
- 给 evaluation harness 增加 LLM-as-a-judge。
- 构建一个包含 answerable 和 unanswerable questions 的 RAG eval set。

## 简历 Talking Points

- 构建 minimal agent framework，包含 explicit state、tool registry、schema validation、step budget 和 trajectory logging。
- 构建 evaluation harness，包含 deterministic checks、latency/cost budgets 和 JSON report export。
