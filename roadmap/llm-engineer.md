# LLM Engineer Roadmap

This roadmap is designed for candidates preparing for LLM engineer, generative AI engineer, and applied AI engineer roles.

## Core Foundations

### Transformer

- Self-attention, multi-head attention, positional encoding.
- Encoder-only, decoder-only, encoder-decoder architectures.
- Causal language modeling and masked language modeling.
- Residual connections, layer normalization, feed-forward networks.

### Training

- Pretraining objective.
- Supervised fine-tuning.
- Instruction tuning.
- RLHF, DPO, and preference optimization.
- Data quality, deduplication, contamination, and synthetic data.

### Inference

- KV cache.
- Prefill and decode stages.
- Batch inference.
- Quantization.
- Speculative decoding.
- Serving latency and throughput tradeoffs.

## Applied LLM Engineering

### Prompting

- Zero-shot, few-shot, chain-of-thought, structured output.
- Prompt injection and prompt leakage.
- Prompt versioning and evaluation.

### RAG

- Document parsing and chunking.
- Embedding models.
- Dense retrieval, sparse retrieval, hybrid retrieval.
- Reranking and query rewriting.
- Grounded generation and citation.

### Agent Systems

- Tool calling.
- Planning and execution.
- Memory.
- Multi-agent workflows.
- Verification and recovery loops.

## Evaluation

- Offline evaluation datasets.
- Golden answers and rubrics.
- LLM-as-a-judge.
- Faithfulness, relevance, latency, cost, safety.
- Regression tests for prompts and workflows.

## Interview Checklist

- Explain Transformer attention from first principles.
- Explain why KV cache improves decoding efficiency.
- Design a RAG system for enterprise documents.
- Compare fine-tuning and RAG.
- Debug hallucination in a production LLM application.
- Design evaluation for an AI agent.
- Discuss cost and latency optimization.
