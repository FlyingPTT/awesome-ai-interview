# Weekly AI Interview Digest: 2026 Week 25

This first digest is a repository-maintenance issue rather than a news issue. It summarizes what candidates should focus on after the initial handbook launch.

## Top Trends

- AI agent engineering is becoming a core interview topic, especially tool use, state, verification, and safety.
- RAG interviews are shifting from "vector database basics" to evaluation, reranking, access control, and production debugging.
- Runnable portfolio projects are more convincing than static notes.

## Model And Product Updates

| Item | Why It Matters | Interview Angle |
| --- | --- | --- |
| Coding agents | They expose real tool-use and repository-editing problems. | Discuss sandboxing, tests, patch review, and audit logs. |
| Evaluation harnesses | Teams need regression gates for prompt and model changes. | Discuss rubrics, deterministic checks, judge calibration, latency, and cost. |
| Agent workflows | Production agents need explicit control flow. | Discuss state machines, tool schemas, stop conditions, and human approval. |

## Papers To Read

| Paper Area | Topic | Takeaway |

## Interview Questions Added

- How do you design a minimal agent loop?
- How do you build an LLM evaluation harness?
- How do you separate retrieval failures from generation failures in RAG?
- How do you prevent an agent from running unsafe tools?

## Project Ideas

- Extend the minimal agent framework with human approval hooks.
- Add LLM-as-a-judge to the evaluation harness.
- Build a small RAG eval set with answerable and unanswerable questions.
- Build a local RAG mini system over Markdown files.
- Build a model router to explain cost, latency, and risk tradeoffs.

## Resume Talking Points

- Built a minimal agent framework with explicit state, tool registry, schema validation, step budget, and trajectory logging.
- Built an evaluation harness with deterministic checks, latency/cost budgets, and JSON report export.
- Built a dependency-free RAG mini system with retrieval scores, citations, and refusal behavior.
- Built a model router that chooses models by risk and complexity while exposing cost and latency estimates.
